### project description: 
An end-to-end data pipeline system that ingests data via Apache Kafka, processes and transforms data using PySpark DataFrames/SparkSQL, and orchestrates the entire workflow via Apache Airflow.

# Stream Analytics Platform – Grading Rubric


This rubric evaluates student projects for the Stream Analytics Platform, emphasizing **architecture design, PySpark transformations, data analysis, schema design, and the full data lifecycle**. The demo section explicitly includes stating **clear objectives** and introducing all datasets, fields, and topics covered.

| Category | Max Points | % Weight | Criteria / Description | Scoring Guidelines |
|----------|-----------|----------|----------------------|-----------------|
| **1. Architecture Design & System Rationale** | 30 | 30% | Clarity of system design, component interactions, justification of technical decisions; inclusion of **data lifecycle stages** (Ingest → Store → Transform → Validate → Analyze → Serve); optional mapping to Bronze/Silver/Gold layers; documentation of micro-batch pattern; **Gold layer schema design (star or snowflake)** | **Exceeds (30 pts):** Complete diagram showing end-to-end flow; all lifecycle stages annotated; rationale for each component; design supports scalability and fault tolerance; Gold layer schema is well-designed (star or snowflake).<br>**Meets (20–29 pts):** Diagram present; some lifecycle stages missing or partially documented; minor justification gaps; Gold layer schema partially considered.<br>**Needs Improvement (<20 pts):** Incomplete/unclear architecture; missing lifecycle mapping; Gold layer schema absent or poorly designed. |
| **2. PySpark Transformation Engine (Module B)** | 25 | 25% | DataFrame/SparkSQL transformations; joins, aggregations, set operations; semi-structured data parsing; optimization (partitioning, caching); parameterized execution; transformations support **Transform → Validate → Analyze** stages; **output conforms to star/snowflake schema in Gold stage** | **Exceeds (25 pts):** Multi-stage transformations implemented; window functions, complex joins, JSON flattening; optimized for performance; parameterized for different datasets; outputs ready for analysis; Gold stage output matches good schema design.<br>**Meets (16–24 pts):** Some transformations missing or partial optimization; limited parameterization; Gold stage schema partially implemented.<br>**Needs Improvement (<16 pts):** Minimal transformations; errors; hard-coded paths; Gold stage schema missing or incorrect. |
| **3. Data Analysis, Insights & End-User Relevance** | 20 | 20% | Analysis answers posed questions; actionable conclusions; connection to defined personas and business problems; uses **Serve stage outputs** effectively | **Exceeds (20 pts):** Comprehensive analysis; insights actionable and tied to personas; clear visualizations or summary tables; demonstrates value for end-users.<br>**Meets (14–19 pts):** Analysis answers some questions; partial insights; limited persona mapping.<br>**Needs Improvement (<14 pts):** Superficial or missing analysis; no actionable conclusions; unclear end-user value. |
| **4. Testing & Data Validation** | 10 | 10% | Unit tests, integration tests, and data validation to ensure **Validate stage** correctness (schema checks, missing/null data, consistency, aggregation correctness) | **Exceeds (10 pts):** Comprehensive tests and validations implemented; automated where possible; ensures accuracy of transformations and final outputs.<br>**Meets (7–9 pts):** Some tests/validation; partial coverage.<br>**Needs Improvement (<7 pts):** Minimal or no tests/validation; errors likely undetected. |
| **5. Airflow Orchestration (Module C)** | 5 | 5% | DAG creation, task dependencies, error handling, backfill support; orchestrates **Ingest → Transform → Validate ** stages | **Exceeds (5 pts):** DAG executes all tasks correctly; dependencies, retries, monitoring implemented; supports backfill; lifecycle stages orchestrated properly.<br>**Meets (4 pts):** DAG mostly functional; minor issues; partial lifecycle orchestration.<br>**Needs Improvement (<4 pts):** DAG incomplete; fails execution; missing lifecycle support. |
| **6. Kafka Streaming / Ingestion Layer (Module A)** | 5 | 5% | Basic ingestion functionality; topic/schema definition; batch consumer implementation; supports **Ingest stage** | **Exceeds (5 pts):** Topics defined; consumer writes valid JSON; ingestion aligns with pipeline needs; basic error handling implemented.<br>**Meets (4 pts):** Consumer works; minor issues; partial documentation.<br>**Needs Improvement (<4 pts):** Consumer fails; missing schema; no alignment with pipeline ingestion. |
| **7. Code Quality & Documentation** | 5 | 5% | Readability, modularity, comments, docstrings, error handling, setup instructions; lifecycle stages clearly documented; Gold schema design documented | **Exceeds (5 pts):** Clean, modular, readable; robust error handling; setup instructions complete; lifecycle stages documented; Gold schema explained.<br>**Meets (4 pts):** Minor readability/documentation issues; partial lifecycle documentation; schema explanation partial.<br>**Needs Improvement (<4 pts):** Poorly organized; unclear or missing documentation; Gold schema not explained. |
| **8. Parameterization & Reproducibility** | 5 | 5% | Configurable inputs/outputs, date ranges, backfill; reproducible runs covering **all lifecycle stages** | **Exceeds (5 pts):** Fully parameterized; reproducible runs; supports historical backfill.<br>**Meets (4 pts):** Partial parameterization; limited reproducibility.<br>**Needs Improvement (<4 pts):** Hard-coded paths/dates; non-reproducible. |
| **9. Demo & Deliverables** | 5 | 5% | End-to-end demo; presentation of pipeline & analysis; diagrams; setup instructions; demonstrates **all data lifecycle stages**; **states clear objectives**; **introduces datasets, fields, and topics covered** | **Exceeds (5 pts):** Seamless demo; clear objectives stated at start; all datasets, fields, and topics introduced; diagrams show lifecycle stages; analysis findings presented clearly.<br>**Meets (4 pts):** Demo works; objectives stated but not fully clear; partial data introduction; lifecycle stages visualized partially.<br>**Needs Improvement (<4 pts):** Demo fails; objectives missing; datasets/fields/topics not introduced; lifecycle stages unclear. |

**Total Points:** 100
